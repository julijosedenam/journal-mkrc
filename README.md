# journal-mkrc

## 项目简介

`journal-mkrc` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面集合的存储与展示空间。每个页面均为独立的 HTML 文件，可单独访问，适合用于记录、存档或分享各类内容。

## 目录说明

```
journal-mkrc/
├── index.html          # 仓库首页，可能包含页面索引
├── pages/              # 存放主要 HTML 页面文件的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/             # 静态资源（样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `index.html`：仓库根目录下的入口页面，通常包含页面列表或导航。
- `pages/`：主要页面文件存放位置，每个 `.html` 文件为一个独立归档项。
- `assets/`：存放页面共用的资源文件，如样式表、JavaScript 脚本或图片。

## 页面归档说明

本仓库中的每个 HTML 页面均为独立归档内容，不依赖外部动态服务。页面可包含文本、图片、样式及交互脚本，但所有资源均本地引用或通过相对路径指向 `assets/` 目录。归档内容不涉及具体域名或营销信息，仅作为静态记录存在。

## 维护说明

- 添加新页面：在 `pages/` 目录下创建新的 `.html` 文件，并确保资源引用路径正确。
- 更新页面：直接修改对应 HTML 文件即可，建议保持文件结构清晰。
- 删除页面：移除对应文件，并更新 `index.html` 中的索引（如有）。
- 资源管理：新增资源文件请放入 `assets/` 对应子目录，避免路径混乱。
